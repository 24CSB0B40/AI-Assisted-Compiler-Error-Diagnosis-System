***SYNTAX ERRORS & PARSING ERRORS***

=> SYNTAX   -> Grammar rules of the language
 		    ->If grammar rules are violated → syntax error

=>PARSER   -> A parser  :
 					takes tokens from the lexer
 					tries to match with the grammar rules [ ** if matching fails -> parsing error ** ]
 					builds a parse tree / AST

=> syntax error 	-> syntax errors occur when the sequence of tokens violates the grammar rules of the programming language.
*** examples ***
 ->Missing semicolon
 ->Unmatched braces
 ->Incorrect keyword usage ( innt x; )
 ->Incorrect statement structure ( int x = ; )

=> parsing errors     -> a parsing error is a failure of the parser to construct a valid parse tree from the given tokens.

  ** Syntax error and parsing error are detected by the SAME PHASE
They differ in perspective, not phase. **
